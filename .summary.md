## Objective
- Deploy PrintHub full-stack app (Spring Boot backend on Railway, Next.js frontend on Vercel) and make them communicate successfully.

## Important Details
- Backend Railway URL: `https://xershop-production.up.railway.app`
- Frontend Vercel URL: `https://xershop-klti383xs-nexusstudios.vercel.app`
- CORS is correctly configured on the backend
- The backend health check and API endpoints respond correctly with proper CORS headers

## Root Cause of the "Network Error"
**Vercel Authentication (Deployment Protection) is enabled** on the Vercel project. This causes:
- All routes (`/`, `/register`, `/login`, etc.) to redirect to `vercel.com/login` before any PrintHub app code runs
- The browser never executes the `authApi.register()` call because it's being redirected to Vercel's login page
- The user sees a "network error" in the browser
